CAB API Notes
==============

Base URL: https://cab.brown.edu/api/

Two main endpoints:

1. Search Endpoint (route=search)
   URL: https://cab.brown.edu/api/?page=fose&route=search&is_ind_study=N&is_canc=N
   Method: POST
   Payload:
     {
       "other": {"srcdb": "<year><semester>"},
       "criteria": [
         {"field": "is_ind_study", "value": "N"},
         {"field": "is_canc", "value": "N"}
       ]
     }
   Semester codes: 00 = Summer, 10 = Fall, 20 = Spring, 15 = Winter
   Example srcdb: "202620" = Spring 2026

   Returns fields per course:
     key, code, title, crn, no (section), total, schd, stat, hide,
     isCancelled, meets, mpkey, meetingTimes, instr, start_date, end_date,
     permreq, rpt, cart_opts, linked_crns, rfam, lrfam, srcdb, changehash

   Does NOT include: description, seats, instructor detail HTML,
   meeting HTML, registration restrictions, class notes, or any other
   rich detail fields.

2. Details Endpoint (route=details)
   URL: https://cab.brown.edu/api/?page=fose&route=details
   Method: POST
   Payload:
     {
       "group": "code:<course_code>",
       "key": "crn:<crn>",
       "srcdb": "<srcdb>"
     }

   Returns all search fields plus:
     description, seats, instructordetail_html, meeting_html,
     credits_html, registration_options, instructor_override,
     registration_restrictions, clssnotes, attr_html,
     resources_critical_review_html, resources_syllabus_html,
     resources_materials_html, exam_html, regdemog_html, regdemog_json,
     all_sections, suggestedcourses_html, books_btn, allInGroup,
     termFlags, section, dbtime, linked_crns, changehash

   This is the ONLY place to get course descriptions.

Required Headers
----------------
The API returns HTTP 202 with an empty body if browser-like headers are
missing. At minimum these are needed:

  accept: application/json, text/javascript, */*; q=0.01
  content-type: application/json
  user-agent: Mozilla/5.0 ...
  x-requested-with: XMLHttpRequest
  origin: https://cab.brown.edu
  referer: https://cab.brown.edu/

Known Issues & Fixes
--------------------
- getCoursesFancyNoFail.py originally wrapped the payload in a "searchData"
  key which caused the API to return {"fatal":"searchData.other is undefined"}.
  Fixed by using the flat payload format matching getCourseInfo.py.

- Python 3.10 on macOS has SSL certificate verification failures with aiohttp.
  Fixed by using ssl.create_default_context(cafile=certifi.where()) and passing
  it to aiohttp.TCPConnector(ssl=ssl_ctx).

- Launching 130k+ concurrent detail requests via asyncio.gather causes timeouts
  and connection exhaustion (~93% completion before crash). The script needs a
  concurrency semaphore (e.g., asyncio.Semaphore) to limit parallel requests
  to a reasonable number (~50-100).

Data Volume
-----------
- ~131,000 course section records across all semesters from Summer 2016
  to Spring 2026.
- Semester counts range from ~10 (winter) to ~6,800 (fall/spring in recent years).
- 8,344 courses have linked_crns (lab/discussion co-enrollment links).

Course Relationships
--------------------
- linked_crns: connects sections requiring co-enrollment (lecture + lab).
- rfam / lrfam: registration family IDs grouping related sections.
- allInGroup: all sections sharing the same course code in a semester.
- all_sections: every section for the course.
- suggestedcourses_html: HTML with recommended related courses.
- description links: <a> tags in description HTML linking to prerequisites
  and related courses (parseCourses.py counts these as num_links).

Cross-references exist at both the section level (linked CRNs, families)
and the content level (description links, suggested courses).

Pipeline
--------
1. getCoursesFancyNoFail.py - Async fetch of all course listings + details
   Output: courses_overview.json (listings), courses.json (details)
2. parseCourses.py - Parse HTML fields into clean text, extract to courseData.tsv
